2.Solvespace, Siemens NX, Onshape 或其他？各有什麼優缺點？ 

【SolveSpace】

優點:擁有浮動調色板視窗(另外的視窗使得繪圖視窗清楚明瞭)、操作簡單容易上手、尺寸標註很清楚也很好修改、3D模型有深淺不同的陰影(增強視覺層次感)。

缺點:定位要注意原點，不然容易出現混淆、功能性單一。

【Siemens NX】

優點:國際化、可以模擬銑切過程和刀具運行狀況、支援匯入刀具且可更改刀具參數、建模很方便、可以捏各種曲面。

缺點:容易讓電腦當機(硬體設備)、功能複雜、設計流程與一般繪圖軟體不同、下載費用較高。

【Onshape】

優點:不須安裝，可直接在線上編輯、自動儲存檔案，檔案不易損毀、可以多人共同編輯(適合團隊合作)。

缺點:零件多時操作會比較吃力、功能NX和SolidWorks較少、介面設計不好、需要穩定網路。

【Inventor】

優點:新手教學影片多(適合新手)、功能介紹清楚、Autodesk工具的整合功能很方便、裝配時有干涉一目了然。

缺點:沒有像NX一樣有這麼多功能(ex:模擬刀具的功能比較弱)、大型複雜的機構裝配比較困難。

【Solidworks】

優點:跟inventor相似度高、可以零件分析模擬(流力、應力)、業界很多公司用。

缺點:檔案容易壞掉、模擬路徑容易出錯、沒辦法多人同時編輯、只能在Windows系統使用。

【總結建議】每款軟體都各有特色，選擇適合的軟體主要取決於你的需求：

  • 若是初學者，可以從 Inventor 或 SolveSpace 開始，這些工具比較簡單易用。

  • 如果需要高度專業功能，尤其是模擬和複雜設計，Siemens NX 和 SolidWorks 是不錯的選擇。

  • 如果需要協作設計，並且偏好雲端解決方案，Onshape 是非常適合的工具。

5-bar Linkage Plotter 與 Openduck mini 零組件設計繪製完成之後，如何切割零件? 為何要切割零件？ 如何利用 Webots 進行後續的控制系統設計與動態模擬 (Dynamic Simulation)？

問題總結:

1.如何切割零件? 

零組件設計完成後可透過草圖切割、旋轉切割、實體分割或布林運算等方式進行切割。

2.為何要切割零件？

為了接下來可以把零件分開放進webots的軟體進行後續的模擬處理，如果是放進整個機構的話，可能讓後面的模擬步驟比起一個一個機件逐步放入更加繁瑣。

3.如何利用 Webots 進行後續的控制系統設計與動態模擬 (Dynamic Simulation)？

建立一個空間，再加入一個名為robot的機器人，用以建立空間樹。接著加入實體、形狀、顏色，接著導入自行設計的零件檔們(利用Mesh放入)，再配合python程式碼，機器人就可以動起來了。

W10

Webots 是什麼？有什麼特點？ 如何進行場景模擬？ 如何用 Python 進行控制系統設計？

Webots 是一套機器人模擬軟體，可在虛擬 3D 環境中進行機器人設計與測試。
它具備真實物理模擬、支援多種程式語言、內建多種機器人模型及感測器等特點。
使用者可建立模擬場景、配置機器人與環境，並透過 Python 撰寫控制程式，讀取感測器資料並控制致動器，以完成控制系統設計與驗證。

W14

問題一 : Webots 的模型有沒有辦法用程式方法建立?

有兩種方法:
方式一：使用 Webots 的 PROTO（參數化模型）

PROTO 是什麼？

  • Webots 的「模型描述語言」

  • 類似「可重複使用的機器人／物件模板」

  • 可用參數改變尺寸、顏色、感測器配置

特點：

  • 不是用 Python 動態生成，而是「文字定義」

  • 適合建立標準化、可重用模型

適合用在：

  • 輪型機器人

  • 固定結構元件

  • 教學或大量測試不同尺寸

       優點：結構清楚、可重複使用
       限制：無法在模擬執行中即時改變結構


方式二：用 Supervisor（監督者）程式動態建立／修改模型

Supervisor 是什麼？

  • Webots 的「高權限控制器」

  • 可用 Python 在模擬中：

    • 新增物件

    • 刪除物件

    • 改變位置、大小、材質

可以做到：

  • 動態生成障礙物

  • 隨機場景配置

  • 研究演算法用的測試環境

       這是真正的「程式生成模型」

方法 是否程式化 適合情境
PROTO 半程式（參數化） 固定結構模型
Supervisor 完全程式化 動態場景、研究用途

問題二 : 有沒有辦法直接從 CAD 套件中轉出 Webots 的場景模型?

可以，但不能一鍵完成，需要轉換流程。Webots 支援由 CAD 套件匯出 STEP、STL、OBJ 等格式匯入場景中，但需手動設定物理屬性、關節與感測器。
因此在實務上常採用 CAD 與 Webots 混合建模的方式，以兼顧外觀精度與物理模擬效果。

Length = 20

平面五連桿 Plotter 設計目標是針對一組 Length cm×Length cm 的繪圖區域，尋找並最佳化五連桿機構中的三個核心設計變數：連桿 L1、連桿 L2，以及底部距離 distance。最終目的是在確保機構穩定運行的前提下，找到能使連桿尺寸最小化（即材料成本與運動慣性最低）的設計。

一、幾何參數設定

    馬達間距 (LENGTH): 兩側連桿（L1 和 L4）的固定轉軸中心必須保持 Length cm 的水平間距。

    繪圖區域 (W×H): 機構必須能覆蓋一個 Length cm 寬度 (W) 和 Length cm 高度 (H) 的完整矩形區域。

    對稱性要求: 機構必須保持對稱，即 L1=L4 且 L2=L3。

二、最小化優化目標 (Objective)

設計應追求連桿總尺寸的最小化。優化目標 J 定義為：
最小化 J=1.0⋅L1+1.0⋅L2+0.5⋅distance

（此權重分配旨在優先最小化連桿長度 L1 和 L2，同時適度控制機構的底部高度 distance。）

三、運動學與工程約束 (Constraints)

設計結果必須嚴格滿足以下三個工程約束，以確保筆尖 C 點在整個繪圖區域內不會遇到死點或不可達的情況。

1. 可達性與外側奇異點迴避 (L1+L2 約束)

連桿 L1 與 L2 的長度總和必須大於機構在繪圖區域最遠點（即 X=Length,Y=distance+Length）處所需的最大距離 Dmax。

這項約束保證了筆尖在最遠處不會發生連桿 L1 和 L2 完全伸直共線（外側奇異點，即 A,B,C 共線）導致的鎖死。

2. 繪圖區域穩定性 (內側奇異點迴避)

連桿 L2 的兩倍長度 (2⋅L2) 必須大於筆尖 C 點在整個 Length cm×Length cm 繪圖區域內時，點 B 到點 D 之間的最大距離 Max(DBD)Area。

更重要的是，為了考慮實際製造公差和控制誤差，必須強制加入 1.0 cm 的安全裕度 (Δ)：
2⋅L2>Max(DBD)Area+1.0 cm

這項約束保證了筆尖在任何位置都不會發生連桿 L2 和 L3 完全伸直共線（內側奇異點，即 B,C,D 共線）導致的控制失效。

3. 轉軸方向約束 (Y 座標約束)

在求解逆運動學時，連桿 L1 和 L4 的擺動點 B 點和 D 點的垂直坐標 (YB 和 YD) 必須始終大於或等於零。

這確保了機構在整個工作過程中始終處於轉軸上方（上搖桿解）的穩定配置，防止機構翻轉至下方區域。

最小尺寸設計如下:

【基因演算法最終最小化尺寸設計結果】
找到**滿足約束且最小化尺寸**的設計（總尺寸 J ≈ 54.00 cm）:
L1 (Link 1 & 4) = 19.50cm
L2 (Link 2 & 3) = 25.50 cm
Distance (Y-bottom) = 18 cm

--- 最終設計驗證 ---
1. 最遠點奇異點約束 (L1+L2 > D_max): 45 > 42.94(滿足)
2. 繪圖區域穩定性 (2*L2 > Max(D_BD) + 1.0cm):
   2*L2 = 51.00 cm
   Max(D_BD) + 1.0 cm = 49.79+ 1.00 = 50.79 cm
   **裕度滿足 (2*L2 僅略大於所需值，實現最小化)**
3. Y 座標約束 (Y_B, Y_D >= 0): 在逆運動學中已確保 (滿足)

Onshape 軟體優缺點?

優點:

1.雲端運作、跨平台使用
完全雲端運作，不需要下載安裝本地程式。只要有網路就能從電腦上工作。
自動儲存與版本控制，不怕版本混亂或檔案覆蓋問題。

2.即時協作與版本管理
支援多人同時編輯與協作。
雲端架構省去獨立 PDM（產品資料管理）系統的維護，團隊合作更省時。

缺點:

1.完全依賴網路
需要穩定的網路連線；沒有正常離線模式。這對網路不穩定或常出差的使用者是大問題。

2.專業功能不如傳統桌面 CAD 完整
與 SolidWorks、Fusion 360 等軟體比起來，某些高級功能較不成熟或欠缺。

這是我第一次使用Onshape這個軟體，跟之前使用的 SOLIDWORKS、Inventor 有些不同，需要重新摸索軟體一陣子，在摸索的過程中也發現了優缺點。我們使用的方法是從副本建立零件工程圖，標註尺寸，雖然過程很麻煩，但這樣不僅可以詳細知道尺寸也可以看三視圖，方便之後的繪製，繪製完所有零件後，再一起進行組裝。
